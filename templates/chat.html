# CẢI TIẾN CHATBOT – STREAMING TOKEN & KIẾN TRÚC PHẢN HỒI Tài liệu này giúp bạn **đọc nhanh – hiểu đúng – làm từng bước** để nâng cấp chatbot hiện tại từ *typing giả lập* sang *streaming token chuẩn LLM*. --- ## 1. Tình trạng hiện tại (AS-IS) ### Backend - Trả về **FULL response** dạng JSON - Không có streaming (SSE / fetch stream / WebSocket) ### Frontend - Có **typing effect** (theo *từ* hoặc *ký tự*) - Chỉ là **giả lập animation**, không phải token thật ### Đánh giá - UI đẹp, mượt, demo tốt - **Chưa đúng kiến trúc LLM production** --- ## 2. Mục tiêu cải tiến (TO-BE) - Backend **stream từng token / chunk** ngay khi LLM sinh - Frontend **render ngay khi nhận** (không đợi full câu) - Tách rõ: - Text thường → streaming - Bảng / Toán / Markdown phức tạp → render 1 lần --- ## 3. Những việc CẦN làm (ưu tiên theo thứ tự) ### (1) Thêm streaming thật ở Backend ⭐⭐⭐⭐⭐ **Mục tiêu**: giảm độ trễ cảm nhận, giống ChatGPT Hướng triển khai: - FastAPI: - `StreamingResponse` - `yield chunk` - Giao thức: - SSE (khuyên dùng) - hoặc fetch stream > Backend chỉ làm 1 việc: **nhận token → đẩy token** --- ### (2) Tách 2 chế độ render rõ ràng ⭐⭐⭐⭐ | Loại nội dung | Cách render | |--------------|-------------| | Text thường | Streaming token | | Bảng | Render 1 lần | | LaTeX | Render 1 lần | | Markdown phức tạp | Render 1 lần | > Không để frontend tự đoán nội dung --- ### (3) Gửi metadata từ Backend ⭐⭐⭐⭐ Thay vì: ```json { "reply": "..." } ``` Nên dùng: ```json { "mode": "stream", "content_type": "text" } ``` Hoặc với SSE: ```text event: meta data: {"content_type":"text"} ``` --- ### (4) Giảm delay giả ở Frontend ⭐⭐⭐ - Bỏ `setTimeout` giả khi đã stream thật - Delay = tốc độ token từ backend - Typing indicator chỉ hiển thị **trước token đầu tiên** --- ## 4. Những việc KHÔNG cần làm ❌ Không cần sửa UI / CSS ❌ Không cần thay Markdown parser ❌ Không cần đụng MathJax ❌ Không cần viết lại frontend toàn bộ > Phần UI của bạn **đã vượt mức đồ án** --- ## 5. Lộ trình đề xuất ### Mức 3 – Chuẩn production 🚀 - Token streaming - Abort / cancel response - Auto-scroll thông minh - Đo first-token latency --- ## 6. Câu mô tả để ghi vào báo cáo > *Hệ thống chatbot được cải tiến bằng cách áp dụng cơ chế streaming theo token ở phía backend nhằm giảm độ trễ cảm nhận, trong khi frontend xử lý hiển thị theo thời gian thực để nâng cao trải nghiệm người dùng.* --- ## 7.
